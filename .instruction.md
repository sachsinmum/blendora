# 🧾 Blendora – Product Requirements Document (PRD)

## 📌 1. Overview

**Product Name:** Blendora
**Type:** Hyperlocal Custom Food Processing Platform
**Platforms:** React Native (Mobile), Web Admin
**Backend:** Spring Boot (Microservices-ready)

### 🎯 Vision

Blendora enables users to **create customized food blends** (Atta, Masala, Oil) and get them freshly processed and delivered via local vendors.

---

## 🎯 2. Objectives

* Build a scalable **multi-category mix platform**
* Enable **custom blend creation**
* Digitize **local food processing vendors**
* Establish **subscription-based recurring revenue**
* Expand beyond atta to **masala and oil**

---

## 👥 3. User Roles

### 3.1 Customer

* Register/Login (OTP आधारित)
* Create custom blends
* Place orders
* Subscribe to recurring deliveries
* Track orders

### 3.2 Vendor (Chakki / Processor)

* Accept/reject orders
* Manage processing queue
* Update order status
* Manage availability

### 3.3 Admin

* Onboard vendors
* Manage ingredients & categories
* Monitor orders
* Configure pricing & commission

---

## 🧩 4. Core Features

### 4.1 Mix Builder Engine (Core USP)

Users can:

* Select category:

  * Atta
  * Masala
  * Oil
* Add multiple ingredients
* Define quantity (kg / %)
* View total composition
* Save mix for reuse

---

### 4.2 Categories

#### Atta

* Wheat, Jowar, Bajra, Chana, etc.

#### Masala

* Turmeric, Chili, Coriander, etc.

#### Oil

* Mustard, Groundnut, Coconut, etc.

---

### 4.3 Order Management

* Add mix to cart
* Select delivery address
* Choose delivery slot
* Place order
* Track status:

  * CREATED
  * ACCEPTED
  * IN_PROGRESS
  * READY
  * DELIVERED

---

### 4.4 Subscription System

* Weekly / Monthly delivery
* Auto-order generation
* Pause/Resume subscription

---

### 4.5 Vendor Matching

* Assign nearest vendor based on:

  * Category support
  * Location proximity
  * Availability

---

### 4.6 Nutrition Calculation (Phase 2)

* Calculate:

  * Protein
  * Fiber
  * Fat
  * Calories

---

### 4.7 Saved Mixes

* Users can:

  * Save custom blends
  * Reorder quickly

---

### 4.8 Notifications

* Order updates (Push/SMS)
* Subscription reminders

---

* OTP-based login
* JWT authentication
* Role-based access control
* Secure APIs
* Audit logging for all requests

---

## 🔁 8. Order Flow

1. User creates mix
2. Adds to cart
3. Places order
4. System assigns vendor
5. Vendor accepts
6. Processing starts
7. Order delivered

---

## 📡 9. API Requirements

### Auth APIs

* POST /auth/send-otp
* POST /auth/verify-otp

### Mix APIs

* POST /mix/create
* GET /mix/user/{id}
* PUT /mix/update

### Order APIs

* POST /order/place
* GET /order/{id}
* PUT /order/status

### Vendor APIs

* GET /vendor/nearby
* PUT /vendor/status
